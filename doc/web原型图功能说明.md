# 智慧宿舍（SmartDorm）— 学生宿舍管理系统 功能说明

---

## 一、系统概述

**智慧宿舍**是一套面向高校的学生宿舍管理系统，采用 B/S 架构，支持移动端（学生端）和桌面端（管理端）两种使用场景。系统核心目标是实现宿舍分配、报修管理、通知公告、选宿批次等业务的数字化和流程化。

### 1.1 角色定义

| 角色 | 标识 | 典型用户 | 主要使用端 |
|------|------|----------|-----------|
| 学生 | 学号 | 在校学生 | 移动端优先（手机浏览器） |
| 管理员 | 工号 | 宿管老师、后勤管理人员 | 桌面端优先（PC 浏览器） |

### 1.2 核心流程

1. **注册**：新用户选择"学生注册"或"老师注册"标签页，填写学号/工号、手机号、密码完成注册。
2. **登录**：输入学号/工号 + 密码 + 图形验证码，后端根据账号类型判断角色，跳转至对应首页。
3. **学生端首页**：底部 Tab 导航，包含"首页（我的宿舍）"、"故障报修"、"我的宿舍"、"费用缴纳"等入口。
4. **管理端首页**：左侧边栏导航，包含"仪表盘"、"宿舍分配"、"报修管理"、"通知公告"、"批次管理"、"系统设置"等模块。
5. **权限校验**：管理端每个页面均需校验登录状态，未登录强制跳转至登录页。

## 七、状态流转

### 7.1 报修工单状态

```
新建
  │
  ├─→ 待分配 ─→ 维修中 ─→ 已完成
  │              │
  │              └─→ 紧急处理 ─→ 维修中 ─→ 已完成
  │
  └─→ 驳回（需填写驳回原因）
```

```
创建 ─→ 待开始 ─→ 进行中 ─→ 已结束
           │

```

### 7.3 公告状态

```

  └──────────────────└─→ 编辑后重新发布
```

### 7.4 床位状态

```
空置 ─→ 已分配 ─→ 退宿 ─→ 空置
```

## 八、设计规范

### 8.1 设计系统

- **主色调**：薄荷绿渐变（#49EACE → #006B5C）
- **背景**：浅色毛玻璃效果（glassmorphism）
- **卡片**：半透明白色 + 柔和彩色阴影
- **图标**：Material Symbols Outlined
- **字体**：Manrope（主字体）、Inter（管理端部分页面）

### 8.2 响应式策略

| 端 | 断点 | 布局 |
|----|------|------|
| 学生端 | < 768px | 移动端优先，底部 Tab 导航 |
| 管理端 | ≥ 1024px | 桌面端优先，左侧边栏导航 |
| 管理端（平板） | 768px - 1023px | 侧边栏可折叠 |

### 8.3 交互规范

- 表单提交前进行前端校验，错误信息显示在字段下方
- 操作确认使用二次弹窗（如删除、分配、退宿）
- 数据加载时显示骨架屏或加载动画
- 操作成功/失败使用 Toast 提示
- 列表为空时显示空状态插图和提示文字
