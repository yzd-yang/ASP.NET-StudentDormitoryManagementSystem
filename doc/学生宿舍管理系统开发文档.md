# 智慧宿舍（SmartDorm）— 学生宿舍管理系统 开发文档

---

## 一、项目概述

**智慧宿舍**是一套面向高校的学生宿舍管理系统，采用 B/S 架构，支持移动端（学生端）和桌面端（管理端）两种使用场景。系统核心目标是实现宿舍分配、报修管理、通知公告、选宿批次等业务的数字化和流程化。

项目采用 **ASP.NET Web Forms 4.8 Website 网站模型**（非 Web Application 项目），代码以 `.aspx` + `.aspx.cs` 形式组织，公共数据访问类放在 `App_Code` 目录。

## 三、角色与权限

| 角色 | 标识 | 典型用户 | 使用端 | 登录凭证 |
|------|------|----------|--------|----------|
| 学生 | 学号 | 在校学生 | 移动端优先（手机浏览器） | 学号 + 密码 + 验证码 |

- **超级管理员**：全部权限

## 五、数据库设计

### 5.1 数据库配置

- 数据库系统：MySQL 8.0
- 数据库名：`smartdorm`
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_general_ci`
- 连接字符串（web.config）：

```xml
<connectionStrings>
  <add name="MySQLConnectionString"
       connectionString="Server=localhost;Port=3306;Database=smartdorm;Uid=root;Pwd=123456;CharSet=utf8mb4;"
       providerName="MySql.Data.MySqlClient" />
</connectionStrings>
```

## 六、功能模块详细设计

### 6.1 共用模块

#### 6.1.1 登录页

- **路径**：`/login.aspx`
- **功能**：标签切换"学生登录"/"管理员登录"，分别输入学号/工号 + 密码 + 验证码
- **学生登录**：调用 `UserBLL.LoginStudent`，无床位时跳转选宿批次页
- **管理员登录**：调用 `UserBLL.LoginAdmin`，跳转仪表盘
- **Session**：
  - 学生登录成功：`Session["UserId"]` = 学生ID，`Session["UserNo"]` = 学号，`Session["UserName"]` = 姓名，`Session["Role"]` = "student"
  - 管理员登录成功：`Session["AdminId"]` = 管理员ID，`Session["AdminNo"]` = 工号，`Session["AdminName"]` = 姓名，`Session["AdminRoleName"]` = 角色名，`Session["Role"]` = "admin"
- **验证码**：`checkcode.aspx` 生成，验证码存 `Session["CheckCode"]`

#### 6.1.2 注册页

- **路径**：`/register.aspx`
- **功能**：仅学生注册，填写学号、手机号、密码（6位及以上）
- **校验**：学号唯一性检查、手机号格式、密码强度

#### 6.1.3 重置密码页

- **路径**：`/reset-password.aspx`
- **功能**：两步流程——先验证学号/工号 + 手机号 + 验证码，验证通过后设置新密码

### 6.2 学生端模块

#### 6.2.1 我的宿舍/首页

- **路径**：`/student/home.aspx`
- **功能**（全部动态数据）：
  - Hero 区域：校区、楼栋、房间号、床位号（`DormBLL.GetStudentDormInfo`），未分配时显示"未分配宿舍"
  - 我的室友：当前学生 + 同寝室友列表（`DormBLL.GetRoommates`）
  - 我的报修：当前学生的报修工单列表（`RepairBLL.GetStudentRepairOrders`），显示类型、描述、状态
  - 通知公告：最新5条已发布公告（`NoticeBLL.GetNoticeList`），置顶标绿，点击跳转详情页
  - 快捷工具：报修申请入口
- **新增页面**：`/student/notice-detail.aspx` 公告详情页

#### 6.2.2 故障报修

- **路径**：`/student/repair.aspx`
- **功能**：
  - **提交申请**：选择报修类型、填写描述、期望上门时间、联系电话、上传照片（最多3张）
  - **我的报修**：报修记录列表，支持按状态筛选，点击查看详情

#### 6.2.3 选宿批次

- **路径**：`/student/batch.aspx`
- **功能**：显示可参与的选宿批次列表，状态为"进行中"的批次可点击"进入选宿"

#### 6.2.4 选/抢宿舍

- **路径**：`/student/grab-dorm.aspx`
- **功能**：
  - 倒计时显示（距选宿截止）
  - 筛选条件（空余床位数）
  - 房间卡片列表，显示床位状态（绿色可选/灰色已占）
  - 选择床位后点击"立即抢订"，弹出确认弹窗完成选宿

#### 6.2.5 个人中心

- **路径**：`/student/profile.aspx`
- **功能**：
  - Hero 区域：姓名、学号、宿舍、手机号（`UserBLL.GetStudentById` + `DormBLL.GetStudentDormInfo`）
  - 个人信息卡片：邮箱、紧急联系人、学院（下拉）、专业（下拉，联动学院）、年级、班级（选填）
  - 点击"编辑资料"进入编辑模式，字段可修改，保存/取消无刷新（UpdatePanel）
  - 入住状态栏：根据床位分配显示"已分配"或"未分配"
  - 学院和专业使用 DropDownList，数据来自 `DormBLL.GetColleges` / `DormBLL.GetMajorsByCollege`
  - 编辑状态通过 JS 控制 readonly/disabled，避免 ASP.NET ReadOnly 阻止回发数据

#### 6.2.6 选宿批次

- **路径**：`/student/batch.aspx`
- **功能**：
  - 无床位学生进入时，顶部导航显示"选宿"按钮（有床位则显示"我的宿舍"）
  - 点击"进入选宿"时校验学院/专业/年级是否填写，未填写弹出提示引导去个人中心

### 6.3 管理端模块

#### 6.3.1 控制台/仪表盘

- **路径**：`/admin/dashboard.aspx`
- **功能**：
  - 统计卡片（动态数据）：总房间数、在宿学生、空余床位、今日报修
  - 7日故障报修趋势图（纯CSS柱状图，动态数据）
  - 楼宇入住率分布（进度条，动态数据）
  - 快捷操作面板（处理报修、宿舍分配、发布公告，含动态数字）

#### 6.3.2 宿舍分配管理

- **路径**：`/admin/allocation.aspx`
- **功能**：
  - 筛选：楼栋、房间号（模糊搜索）
  - 操作：批量导入（Excel）、导出表格、集中退宿
  - 房间卡片：显示入住状态（满员/空余X张/全空置）
  - 分配弹窗：搜索学生（学号/姓名）→ 确认分配到指定床位

#### 6.3.3 报修管理

- **路径**：`/admin/repair.aspx`
- **功能**：
  - 统计卡片：待处理、处理中、紧急、今日已完成
  - 筛选：状态、宿舍楼、报修类型
  - 工单列表表格
  - 详情侧边面板：现场照片、学生信息、报修描述、派工处理、内部备注
  - 操作：驳回（需填写原因）/ 确认完成
  - 状态流转：待分配 → 维修中 → 已完成（可经紧急处理）

#### 6.3.4 通知公告管理

- **路径**：`/admin/notice.aspx`
- **功能**：
  - 发布公告：标题、发送范围、公告类别、富文本内容、是否置顶、立即发送/存草稿
  - 已发布公告列表：编辑、撤回、删除操作
  - 分页：每页10条

#### 6.3.5 选宿批次管理

- **路径**：`/admin/batch.aspx`
- **功能**：
  - 统计卡片：总批次、进行中、待开始、已结束
  - 创建批次：批次名称、宿舍范围（楼栋→楼层→房间多选）、开始/结束时间、学院/专业限定
  - 批次列表：编辑、暂停、删除操作
  - 状态筛选：全部/进行中/待开始/已结束

#### 6.3.6 系统管理

- **路径**：`/admin/system.aspx`
- **功能**：
  - **管理员账号管理**：搜索、角色筛选、状态筛选；编辑、重置密码、删除
  - **楼宇管理**：楼宇列表，编辑/删除
  - **批量生成房间**：选择楼宇 → 输入楼层/每层房间数 → 预览 → 批量创建
  - **院系专业管理**：树形结构展示学院→专业，支持增删改

## 十二、编码规范

### 12.1 文件编码

- 所有 `.aspx`、`.aspx.cs`、`.master`、`.master.cs` 文件必须使用 **UTF-8 with BOM** 编码
- BOM 标记（EF BB BF）确保 ASP.NET 正确识别中文字符

### 12.2 Page 指令

- 所有 `.aspx` 页面的 `@ Page` 指令需添加 `ResponseEncoding="utf-8"`
- `@ Master` 指令**不支持** `ResponseEncoding`，母版页编码由 BOM 和 web.config 保证

### 12.3 web.config 全球化配置

```xml
<system.web>
  <globalization requestEncoding="utf-8" responseEncoding="utf-8" fileEncoding="utf-8" />
</system.web>
```

## 十三、注意事项

- `.aspx` 和 `.aspx.cs` 是配对文件，改逻辑必须同时理解两者的关联
- `App_Code/` 下的类是全局可访问的（Website 模型自动编译），新增文件会自动参与
- 学生端母版页提供底部Tab导航，管理端母版页提供左侧边栏导航，两者独立
- 抢宿舍功能需考虑并发控制，使用数据库事务 + 行锁（`SELECT ... FOR UPDATE`）
- 图片上传路径：头像 `Uploads/avatars/`，报修照片 `Uploads/repair/`
- 所有数据库操作使用参数化查询，禁止字符串拼接SQL
- 不需要的功能：费用缴纳、访客管理、操作日志
