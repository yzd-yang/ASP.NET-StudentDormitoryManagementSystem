# 智慧宿舍（SmartDorm）— 学生宿舍管理系统 开发文档

---

## 一、项目概述

**智慧宿舍**是一套面向高校的学生宿舍管理系统，采用 B/S 架构，支持移动端（学生端）和桌面端（管理端）两种使用场景。系统核心目标是实现宿舍分配、报修管理、通知公告、选宿批次等业务的数字化和流程化。

项目采用 **ASP.NET Web Forms 4.8 Website 网站模型**（非 Web Application 项目），代码以 `.aspx` + `.aspx.cs` 形式组织，公共数据访问类放在 `App_Code` 目录。

## 三、角色与权限

| 角色 | 标识 | 典型用户 | 使用端 | 登录凭证 |

| 学生 | 学号 | 在校学生 | 移动端优先（手机浏览器） | 学号 + 密码 + 验证码 |

## 五、数据库设计

### 5.1 数据库配置

- 数据库系统：MySQL 8.0
- 数据库名：`smartdorm`
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_general_ci`
- 连接字符串（web.config）：

```xml
<connectionStrings>
  <add name="MySQLConnectionString"
       connectionString="Server=localhost;Port=3306;Database=smartdorm;Uid=root;Pwd=123456;CharSet=utf8mb4;"
       providerName="MySql.Data.MySqlClient" />
</connectionStrings>
```

## 六、功能模块详细设计

### 6.1 共用模块

#### 6.1.1 登录页

- **路径**：`/login.aspx`
- **功能**：输入学号/工号 + 密码 + 图形验证码，后端根据账号前缀或查询结果判断角色，跳转至对应首页
- **Session**：
  - 学生登录成功：`Session["UserId"]` = 学生ID，`Session["UserNo"]` = 学号，`Session["Role"]` = "student"
  - 管理员登录成功：`Session["AdminId"]` = 管理员ID，`Session["AdminNo"]` = 工号，`Session["Role"]` = "admin"
- **验证码**：`checkcode.aspx` 生成，验证码存 `Session["CheckCode"]`

#### 6.1.2 注册页

- **路径**：`/register.aspx`
- **功能**：标签页切换"学生注册"/"老师注册"，填写学号/工号、手机号、密码
- **校验**：学号/工号唯一性检查、手机号格式、密码强度（8-20位字母数字）

#### 6.1.3 重置密码页

- **路径**：`/reset-password.aspx`
- **功能**：验证学号/工号 + 手机号 + 验证码后，设置新密码

### 6.2 学生端模块

#### 6.2.1 我的宿舍/首页

- **路径**：`/student/home.aspx`
- **功能**：
  - 显示宿舍信息卡片（校区、楼栋、房间号、床位号）
  - 宿舍评分、住宿费缴纳状态
  - 室友列表（头像、姓名、学院、年级）
  - 设施状态（空调、热水器、照明、网络）
  - 快捷工具入口

#### 6.2.2 故障报修

- **路径**：`/student/repair.aspx`
- **功能**：
  - **提交申请**：选择报修类型、填写描述、期望上门时间、联系电话、上传照片（最多3张）
  - **我的报修**：报修记录列表，支持按状态筛选，点击查看详情

#### 6.2.3 选宿批次

- **路径**：`/student/batch.aspx`
- **功能**：显示可参与的选宿批次列表，状态为"进行中"的批次可点击"进入选宿"

#### 6.2.4 选/抢宿舍

- **路径**：`/student/grab-dorm.aspx`
- **功能**：
  - 倒计时显示（距选宿截止）
  - 筛选条件（空余床位数）
  - 房间卡片列表，显示床位状态（绿色可选/灰色已占）
  - 选择床位后点击"立即抢订"，弹出确认弹窗完成选宿

#### 6.2.5 个人中心

- **路径**：`/student/profile.aspx`
- **功能**：查看/编辑个人信息、联系信息、学业信息、入住状态

#### 6.2.6 费用缴纳

- **路径**：`/student/payment.aspx`
- **功能**：查看住宿费缴纳记录、缴费状态

### 6.3 管理端模块

#### 6.3.1 控制台/仪表盘

- **路径**：`/admin/dashboard.aspx`
- **功能**：
  - 统计卡片：总房间数、在宿学生、空余床位、今日报修、待审核访客
  - 图表：7日故障报修趋势（柱状图）、楼宇入住率分布（进度条）
  - 快捷操作入口


#### 6.3.2 宿舍分配管理

- **路径**：`/admin/allocation.aspx`
- **功能**：
  - 筛选：楼栋、房间号（模糊搜索）、网格/列表视图切换
  - 操作：批量导入（Excel）、导出表格、集中退宿
  - 房间卡片：显示入住状态（满员/空余X张/全空置）
  - 分配弹窗：搜索学生（学号/姓名）→ 确认分配到指定床位

#### 6.3.3 报修管理

- **路径**：`/admin/repair.aspx`
- **功能**：
  - 统计卡片：待处理、处理中、紧急、今日已完成
  - 筛选：状态、宿舍楼、报修类型
  - 工单列表表格
  - 详情侧边面板：现场照片、学生信息、报修描述、派工处理、内部备注
  - 操作：驳回（需填写原因）/ 确认完成
  - 状态流转：待分配 → 维修中 → 已完成（可经紧急处理）

#### 6.3.4 通知公告管理

- **路径**：`/admin/notice.aspx`
- **功能**：
  - 发布公告：标题、发送范围、公告类别、富文本内容、是否置顶、立即发送/存草稿
  - 已发布公告列表：编辑、撤回、删除操作
  - 分页：每页10条

#### 6.3.5 选宿批次管理

- **路径**：`/admin/batch.aspx`
- **功能**：
  - 统计卡片：总批次、进行中、待开始、已结束
  - 创建批次：批次名称、宿舍范围（楼栋→楼层→房间多选）、开始/结束时间、学院/专业限定
  - 批次列表：编辑、暂停、删除操作
  - 状态筛选：全部/进行中/待开始/已结束

#### 6.3.6 系统管理

- **路径**：`/admin/system.aspx`
- **功能**：
  - **管理员账号管理**：搜索、角色筛选、状态筛选；编辑、重置密码、删除
  - **楼宇管理**：楼宇列表，编辑/删除
  - **批量生成房间**：选择楼宇 → 输入楼层/每层房间数 → 预览 → 批量创建
  - **院系专业管理**：树形结构展示学院→专业，支持增删改

## 九、前端设计规范

### 9.1 设计系统

- **主色调**：薄荷绿渐变（#49EACE → #006B5C）
- **背景**：浅色毛玻璃效果（glassmorphism）
- **卡片**：半透明白色 + 柔和彩色阴影
- **图标**：Material Symbols Outlined
- **字体**：Manrope（主字体）、Inter（管理端）

### 9.2 响应式策略

| 端 | 断点 | 布局 |
|----|------|------|
| 学生端 | < 768px | 移动端优先，底部 Tab 导航 |
| 管理端 | ≥ 1024px | 桌面端优先，左侧边栏导航 |
| 管理端（平板） | 768px - 1023px | 侧边栏可折叠 |

### 9.3 交互规范

- 表单提交前进行前端校验，错误信息显示在字段下方
- 操作确认使用二次弹窗（如删除、分配、退宿）
- 数据加载时显示骨架屏或加载动画
- 操作成功/失败使用 Toast 提示
- 列表为空时显示空状态插图和提示文字

```
  ├─→ 待分配 ─→ 维修中 ─→ 已完成
  │              │
  │              └─→ 紧急处理 ─→ 维修中 ─→ 已完成
  │
  └─→ 驳回（需填写驳回原因）
```

### 10.2 选宿批次状态

```
           │
           └─→ 暂停（可恢复）
```

```
草稿 ─→ 已发布 ─→ 已撤回
  │                  │
  └──────────────────└─→ 编辑后重新发布
```

### 10.4 床位状态

```
空置 ─→ 已分配 ─→ 退宿 ─→ 空置
```

## 十一、安全设计

| 项目 | 措施 |
|------|------|
| 密码存储 | MD5 哈希（`FormsAuthentication.HashPasswordForStoringInConfigFile`） |
| SQL注入 | 全部使用 `MySqlParameter` 参数化查询 |
| 验证码 | GDI+ 动态生成，Session 校验 |
| 登录校验 | 每个页面 `Page_Load` 检查 Session，未登录跳转登录页 |
| 文件上传 | 限制文件类型（jpg/png）、大小（5MB）、重命名存储 |
| 权限隔离 | 学生端/管理端独立母版页，Session 角色校验 |
| 请求验证 | 敏感页面保持 `ValidateRequest="true"` |

## 十四、注意事项

- `.aspx` 和 `.aspx.cs` 是配对文件，改逻辑必须同时理解两者的关联
- `App_Code/` 下的类是全局可访问的（Website 模型自动编译），新增文件会自动参与
- 学生端母版页提供底部Tab导航，管理端母版页提供左侧边栏导航，两者独立
- 抢宿舍功能需考虑并发控制，使用数据库事务 + 行锁（`SELECT ... FOR UPDATE`）
- 图片上传路径：头像 `Uploads/avatars/`，报修照片 `Uploads/repair/`
- 所有数据库操作使用参数化查询，禁止字符串拼接SQL
