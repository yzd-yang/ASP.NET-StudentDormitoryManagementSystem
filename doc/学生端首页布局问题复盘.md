# 学生端首页布局问题复盘

## 背景

本次调整针对学生端首页 `/student/home.aspx` 的页面布局问题。页面目标是尽量还原 `UI原型图/我的宿舍` 的视觉结构，同时解决顶部导航、主内容区域、移动端导航和样式覆盖等问题。

## 页面异常现象

调整前或浏览器加载旧样式时，学生端首页主要出现以下问题：

1. 顶部导航没有按设计稿显示为固定毛玻璃导航，而是变成普通文字从上到下排列。
2. 搜索、通知按钮显示为浏览器默认按钮样式，和整体 UI 风格不一致。
3. 桌面端底部 Tab 导航错误显示，遮挡页面底部内容。
4. 首页主体布局和原型不一致，原本应为“左侧主内容 + 右侧设施状态栏”的结构，却被做成三张卡片等分排列。
5. 主内容区域没有合理的最大宽度限制，在大屏幕下容易铺得过宽，页面视觉重心不稳定。
6. 首页私有样式与公共样式存在类名冲突，例如 `.roommate-card`、`.facility-list` 等，后续维护时容易互相覆盖。
7. `student.css` 被重复引入，导致样式加载顺序不清晰，增加了布局被覆盖的风险。

## 根因分析

### 1. CSS 重复引入

母版页 `student/MasterPage.master` 已经引入了：

```html
<link href="/css/student.css" rel="stylesheet" />
```

但首页 `student/home.aspx` 又重复引入了一次同一个 CSS 文件。这样会造成后加载的公共 CSS 覆盖页面或母版中的局部样式，导致顶部栏高度、主内容间距等表现不稳定。

### 2. 母版页样式分散

顶部导航样式原本一部分写在 `student.css`，一部分写在 `MasterPage.master` 的内联 `<style>` 中。样式来源分散后，很容易出现维护时漏改、覆盖顺序不明确的问题。

### 3. 首页布局未按原型结构实现

原型中的首页结构是：

```text
Hero 宿舍信息区
├── 左侧主内容：宿舍评分、住宿费缴纳、我的室友
└── 右侧栏：设施状态、快捷工具
```

旧实现将“宿舍评分 / 住宿费缴纳 / 设施状态”做成三等分卡片，室友区域再单独铺满一行。这和原型的信息层级不一致，也导致中等屏幕下容易出现空位和视觉不平衡。

### 4. 移动端导航只有样式没有结构

`css/student.css` 中存在 `.student-tabbar` 的样式，但母版页里没有对应的底部导航 DOM。后续补结构时，如果浏览器仍加载旧 CSS，就会出现桌面端也显示底部导航的问题。

### 5. 浏览器缓存旧版 CSS

修复后截图中仍然出现顶部导航失效、桌面端底部 Tab 显示的问题。该表现与旧版 `student.css` 完全一致，说明浏览器或 IIS 静态资源缓存仍在使用旧 CSS 文件。

## 本次修改内容

### 1. 重构学生端公共布局样式

修改文件：`css/student.css`

主要调整：

1. 将顶部导航 `.student-header`、`.header-left`、`.header-nav`、`.header-right` 等样式统一放入公共 CSS。
2. 统一顶部导航高度为 `64px`，并设置固定定位、毛玻璃背景和底部分隔线。
3. 将 `.student-main` 改为居中容器，最大宽度限制为 `1440px`。
4. 新增 `.student-footer` 相关样式，替代母版页中的大量内联样式。
5. 新增移动端底部 Tab 样式，并设置为仅在小屏幕下显示。
6. 移动端隐藏部分顶部栏元素，避免搜索、通知、用户名称在窄屏下挤压。

### 2. 整理学生端母版页

修改文件：`student/MasterPage.master`

主要调整：

1. 移除母版页中的顶部导航内联样式。
2. 保留 `style.css` 和 `student.css` 两个公共样式入口。
3. 新增真实的移动端底部 Tab 导航结构。
4. 将页脚改为语义化结构，并使用公共 CSS 类控制样式。
5. 为 `student.css` 增加版本号参数，解决浏览器缓存旧样式的问题：

```html
<link href="/css/student.css?v=20260612-home-layout" rel="stylesheet" />
```

### 3. 优化导航激活状态

修改文件：`student/MasterPage.master.cs`

主要调整：

当当前页面为 `grab-dorm.aspx` 时，让顶部和底部导航中的“选宿”保持激活状态，避免用户进入选宿详情页后导航状态丢失。

### 4. 重构学生端首页

修改文件：`student/home.aspx`

主要调整：

1. 移除首页中重复的 `/css/student.css` 引用。
2. 将首页布局改为接近原型的结构：Hero 区、左侧主列、右侧栏。
3. 将“设施状态”移动到右侧栏，不再和评分、缴费三等分排列。
4. 新增“快捷工具”模块，使页面内容层级更接近原型。
5. 首页私有样式统一使用 `home-` 前缀，例如 `.home-card`、`.home-dashboard`、`.home-facility-list`，减少与公共样式冲突。
6. Hero 区使用背景图加渐变兜底，避免图片加载失败时出现破图图标。
7. 针对桌面、平板、移动端分别设置响应式布局。

## 修复后的布局结构

当前首页结构如下：

```text
student/MasterPage.master
├── 固定顶部导航
├── main.student-main
│   └── student/home.aspx
│       ├── Hero 宿舍信息区
│       └── home-dashboard
│           ├── home-main-column
│           │   ├── 宿舍评分
│           │   ├── 住宿费缴纳
│           │   └── 我的室友
│           └── home-side-column
│               ├── 设施状态
│               └── 快捷工具
├── 移动端底部 Tab 导航
└── 页脚
```

## 验证结果

已完成以下静态检查：

1. `git diff --check`：未发现空白或格式错误。
2. 检查 `student.css` 引用：当前只在 `MasterPage.master` 中引入一次。
3. 检查首页类名：首页样式已改为 `home-` 前缀，避免继续覆盖公共 `.roommate-card`、`.facility-list` 等类。
4. 检查移动端底部导航：默认隐藏，仅在 `max-width: 767px` 时显示。

## 后续注意事项

1. 修改公共 CSS 后，如果浏览器仍显示旧效果，优先检查是否存在缓存问题，可通过更新 CSS 版本号或 `Ctrl + F5` 强制刷新解决。
2. 学生端页面不要重复引入 `/css/student.css`，统一由 `MasterPage.master` 引入。
3. 页面私有样式建议使用页面名前缀，例如首页使用 `home-`，个人中心使用 `profile-`，避免和公共组件类名冲突。
4. 母版页中的通用布局样式应尽量放在 `css/student.css` 中，减少内联样式，方便统一维护。
5. 新增学生端页面时，应同时检查桌面端、平板端和移动端布局，尤其是顶部导航和底部 Tab 是否按预期显示。
