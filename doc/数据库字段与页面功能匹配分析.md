# 数据库字段与页面功能匹配分析

## 一、数据库表结构总览

| 表名 | 用途 | 主要字段 |
|------|------|----------|
| `Students` | 学生信息 | Id, StudentNo, Name, Phone, Password, Avatar, College, Major, Grade, ClassName, Email, EmergencyContact, EmergencyRelation, EmergencyPhone, Status, CreateTime |
| `Admins` | 管理员信息 | Id, AdminNo, Name, Phone, Password, Role(1超管/2宿管/3后勤), Status, CreateTime |
| `Buildings` | 楼宇 | Id, Name, FloorCount, RoomsPerFloor, Campus, Status, CreateTime |
| `Rooms` | 房间 | Id, BuildingId, Floor, RoomNo, RoomType(1双/2四/3六), BedCount, Status, CreateTime |
| `Beds` | 床位 | Id, RoomId, BedNo, StudentId(可空), Status(0空/1已分), AllocateTime |
| `RepairOrders` | 报修工单 | Id, OrderNo, StudentId, RoomId, RepairType(1水电/2家具/3网络/4其他), Description, Photos, ExpectTime, ContactPhone, Status(1待分配/2维修中/3紧急/4完成/5驳回), AssignAdminId, InternalNote, RejectReason, CreateTime, CompleteTime |
| `Notices` | 通知公告 | Id, Title, Content, Scope(0全体/1A栋/2B栋/3C栋), Category(1行政/2安全/3生活/4活动), IsTop, Status(0草稿/1已发布/2已撤回), PublishTime, AdminId, CreateTime |
| `SelectionBatches` | 选宿批次 | Id, BatchName, StartTime, EndTime, CollegeLimit(JSON), MajorLimit(JSON), GradeLimit, Status(0待开始/1进行中/2已结束/3暂停), AdminId, CreateTime |
| `BatchRooms` | 批次房间关联 | Id, BatchId, RoomId |
| `FacilityStatus` | 设施状态 | Id, RoomId, FacilityType(1空调/2热水器/3照明/4网络), Status(1正常/2报修中/3故障), UpdateTime |
| `Departments` | 院系专业 | Id, CollegeName, MajorName, SortOrder |

## 二、现有BLL方法汇总

### UserBLL
| 方法 | 功能 | 涉及表 |
|------|------|--------|
| `Login(userNo, password)` | 登录验证 | Students, Admins |
| `RegisterStudent(...)` | 学生注册 | Students |
| `GetStudentById(id)` | 获取学生信息 | Students |
| `UpdateStudentInfo(...)` | 更新学生信息 | Students |
| `GetAdminById(id)` | 获取管理员信息 | Admins |

### DormBLL
| 方法 | 功能 | 涉及表 |
|------|------|--------|
| `GetStudentDormInfo(studentId)` | 获取学生宿舍信息 | Students, Beds, Rooms, Buildings |
| `GetRoommates(studentId)` | 获取室友列表 | Beds, Students |
| `GetBuildings()` | 获取楼宇列表 | Buildings |
| `GetRoomsByBuilding(buildingId)` | 获取房间列表(含入住数) | Rooms, Beds |
| `GetRoomBeds(roomId)` | 获取床位列表(含学生信息) | Beds, Students |
| `AllocateBed(bedId, studentId)` | 分配床位 | Beds |
| `ReleaseBed(bedId)` | 释放床位 | Beds |
| `SearchStudents(keyword)` | 搜索学生 | Students |
| `GetTotalRooms()` | 总房间数 | Rooms |
| `GetTotalStudents()` | 总学生数 | Students |
| `GetAvailableBeds()` | 空余床位数 | Beds |

### RepairBLL
| 方法 | 功能 | 涉及表 |
|------|------|--------|
| `CreateRepairOrder(...)` | 创建报修工单 | RepairOrders |
| `GetStudentRepairOrders(studentId, status)` | 学生的报修列表 | RepairOrders, Rooms, Buildings |
| `GetAllRepairOrders(status)` | 所有报修列表 | RepairOrders, Students, Rooms, Buildings |
| `AssignRepairOrder(orderId, adminId)` | 指派维修工 | RepairOrders |
| `CompleteRepairOrder(orderId)` | 完成工单 | RepairOrders |
| `RejectRepairOrder(orderId, reason)` | 驳回工单 | RepairOrders |
| `GetPendingRepairCount()` | 待处理工单数 | RepairOrders |
| `GetProcessingRepairCount()` | 处理中工单数 | RepairOrders |

## 三、页面功能与数据库匹配分析

### 3.1 公共页面

| 页面功能 | 数据库支持 | 说明 |

## 五、结论

### ✅ 可直接实现的功能（约75%）
- 登录/注册/重置密码（需小量BLL补充）
- 学生端：宿舍信息展示、室友列表、设施状态、报修申请、报修列表、个人中心
- 管理端：宿舍分配、报修管理

### ⚠️ 需要新增BLL方法的功能（约20%）
- 通知公告CRUD → 新建 `NoticeBLL.cs`
- 选宿批次CRUD → 新建 `BatchBLL.cs`
- 管理员管理CRUD → 新建 `AdminBLL.cs`
- 院系专业管理 → 新建 `DeptBLL.cs`

### ❌ 使用静态数据的功能（约5%）
- 宿舍评分（页面已写死94分）
- 仪表盘实时动态（页面已写死静态列表）

> **总结：现有数据库字段可以支撑约95%的页面功能。需要新建4个BLL文件（NoticeBLL、BatchBLL、AdminBLL、DeptBLL）来实现剩余的CRUD操作。宿舍评分和实时动态使用静态数据展示，不需要新建表。**
