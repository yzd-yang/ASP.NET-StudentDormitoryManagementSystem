# 问题及解决方案

## 1. 中文乱码

**现象**：页面中文显示为乱码

**原因**：文件缺少 UTF-8 BOM、Page 指令缺少编码配置、web.config 缺少全球化设置

**解决方案**：

1. web.config 添加全球化配置：
```xml
<system.web>
  <globalization requestEncoding="utf-8" responseEncoding="utf-8" fileEncoding="utf-8" />
</system.web>
```

2. `.aspx` 文件的 `@ Page` 指令添加 `ResponseEncoding="utf-8"`

3. 所有 `.aspx`、`.aspx.cs`、`.master` 文件使用 UTF-8 with BOM 编码

4. 批量添加 BOM：
```powershell
Get-ChildItem -Path '项目路径' -Include '*.aspx','*.aspx.cs','*.master' -Recurse | ForEach-Object {
    $content = [System.IO.File]::ReadAllText($_.FullName, [System.Text.Encoding]::UTF8)
    $utf8bom = New-Object System.Text.UTF8Encoding $true
    [System.IO.File]::WriteAllText($_.FullName, $content, $utf8bom)
}
```

**注意**：`@ Master` 指令不支持 `ResponseEncoding`，母版页编码由 BOM 和 web.config 保证。

## 2. MasterPage 不支持 ResponseEncoding

**现象**：`分析特性"responseencoding"时出错: 类型"System.Web.UI.MasterPage"不具有名为"responseencoding"的公共属性`

**原因**：`@ Master` 指令不支持 `ResponseEncoding` 属性

**解决方案**：从 `.master` 文件的 `@ Master` 指令中移除 `ResponseEncoding="utf-8"`

## 3. 代码引用已删除的服务器控件

**现象**：`CS0103: 当前上下文中不存在名称"xxx"`

**原因**：`.aspx.cs` 中引用了 `.aspx` 页面中已删除的服务器控件（如 Literal、Repeater、Panel 等）

**解决方案**：检查 `.aspx.cs` 文件，移除对不存在控件的引用，或在 `.aspx` 中添加对应的服务器控件

**示例**：
```csharp
// 错误：页面中没有 litDormInfo 控件
litDormInfo.Text = "xxx";

// 修复：移除引用或使用静态数据
```

## 4. C# 7.0 语法不兼容

**现象**：`CS1026: 应输入 )` 或其他编译错误

**原因**：.NET Framework 4.8 默认使用 C# 7.0 以下版本，不支持新语法

**不支持的语法**：

| 语法 | 示例 | 替代方案 |
|------|------|----------|
| 模式匹配 `is` | `if (obj is DataTable dt)` | `DataTable dt = obj as DataTable; if (dt != null)` |
| 模式匹配 `switch` | `case int i:` | 使用 `if-else` + 类型转换 |
| 元组解构 | `var (a, b) = GetTuple();` | 分别访问 Item1、Item2 |
| `out` 变量声明 | `int.TryParse(s, out int i)` | 先声明变量再传入 |

**解决方案**：使用传统类型转换替代模式匹配

```csharp
// 错误
if (rptTrendChart.DataSource is DataTable dt) { ... }

// 正确
DataTable dt = rptTrendChart.DataSource as DataTable;
if (dt != null) { ... }
```

## 5. 查询 BOM 和编码

**检查文件是否有 BOM**：

[System.IO.File]::ReadAllBytes('文件路径')[0..2] -join ','
# 有 BOM: 239,187,191
# 无 BOM: 60,37,64 (即 <)

## 6. 弹窗内联动下拉框回发丢失数据

**现象**：弹窗中选择下拉框后，其他下拉框选项或动态数据消失

**原因**：`AutoPostBack="true"` 触发整页回发，`HtmlSelect` 的动态选项不保留在 ViewState 中，HiddenField 值也可能丢失

**解决方案**：使用 `RegisterStartupScript` 预加载数据到 JS 变量，选择时纯前端读取，不触发回发

```csharp
// Page_Load 中注入数据
string script = "var _bd=" + buildingJson + ";var _md=" + majorJson + ";";
Page.ClientScript.RegisterStartupScript(this.GetType(), "batchData", script, true);
```

```javascript
// JS 中按需读取
function onBuildingChange() {
    var buildingId = ddlBuilding.value;
    if (_bd[buildingId]) {
        _bd[buildingId].floors.forEach(function(f) { /* 填充楼层 */ });
    }
}
```

**不要用**：HiddenField（回发后值丢失）、AutoPostBack（弹窗状态丢失）、AJAX handler（本项目不需要）

## 7. DateTime 解析失败

**现象**：`FormatException: 该字符串未被识别为有效的 DateTime`

**原因**：HTML `<input type="datetime-local">` 返回格式 `2026-03-01T08:00`（带 T 分隔符），`Convert.ToDateTime` 在部分语言环境下无法解析

**解决方案**：使用 `DateTime.ParseExact` 指定格式

```csharp
DateTime startTime = DateTime.ParseExact(txtStartTime.Text, "yyyy-MM-ddTHH:mm",
    System.Globalization.CultureInfo.InvariantCulture);
```

## 8. 刷新页面重复提交

**现象**：保存后按 F5 刷新，数据被重复插入

**原因**：POST 请求后直接渲染页面，浏览器刷新会重发 POST

**解决方案**：POST-Redirect-GET 模式，保存后用 `Response.Redirect` 跳转

```csharp
if (BLL.AddData(...))
{
    Session["ToastMsg"] = "保存成功";
    Response.Redirect(Request.RawUrl); // GET 请求，刷新安全
}
```

Toast 消息通过 Session 传递，页面加载时读取并清除。

## 9. HtmlSelect 与 DropDownList 属性差异

**现象**：`CS1061: 'DropDownList' 不包含 'Value' 的定义`

**原因**：HTML `<select runat="server">`（`HtmlSelect`）用 `.Value` 读取选中值，ASP.NET `<asp:DropDownList>` 用 `.SelectedValue`

**解决方案**：
- `HtmlSelect` → `.Value`
- `DropDownList` → `.SelectedValue`

---

## 10. ASP.NET 事件验证失败

**现象**：`ArgumentException: 回发或回调参数无效`

**原因**：JS 动态添加的下拉选项不在服务端原始渲染的选项中，ASP.NET 事件验证拒绝处理

**解决方案**：在 `@ Page` 指令中添加 `EnableEventValidation="false"`（仅限确实需要 JS 动态修改选项的页面）

## 11. BLL 方法参数遗漏导致业务逻辑不生效

**现象**：页面计算的状态值传不到数据库，新建批次永远是"待开始"

**原因**：`BatchBLL.AddBatch` 的 INSERT SQL 中 `Status` 硬编码为 `0`，方法签名没有 `status` 参数，页面计算的 `batchStatus` 被忽略

**解决方案**：BLL 方法签名加参数，SQL 用参数化占位符

```csharp
// 修复前
public static bool AddBatch(..., int[] roomIds) {
    string sql = "... Status=0 ...";  // 硬编码
}

// 修复后
public static bool AddBatch(..., int[] roomIds, int status) {
    string sql = "... @Status ...";   // 参数化
}
```

**教训**：修改页面逻辑时，同步检查 BLL 方法签名是否支持新参数。硬编码值是最隐蔽的 bug 来源。

## 12. CSS 样式重复前缀导致不生效

**现象**：仪表盘报修趋势图的柱状条没有颜色

**原因**：`GetBarColor` 方法返回 `background:rgba(73,234,206,0.25);`，但 aspx 模板中已有 `background:` 前缀：

```html
<div class="chart-bar" style="height:50%; background:<%# GetBarColor(Eval("Count")) %>;"></div>
```

渲染后变成 `background:background:rgba(73,234,206,0.25);`，无效 CSS。

**解决方案**：`GetBarColor` 只返回颜色值，不带 `background:` 前缀：

```csharp
// 修复前
if (count == 0) return "background:rgba(73,234,206,0.1);";

// 修复后
if (count == 0) return "rgba(73,234,206,0.1)";
```

**教训**：BLL 辅助方法返回的值要与模板中的拼接方式匹配。如果模板已写 `background:`，方法只返回颜色值。

## 13. 学生端选宿舍界面可同时选择多个房间床位

**现象**：每个宿舍卡片都能选择一个床位，学生可以同时在多个房间各选一个床位

**原因**：`selectBed` 函数只在当前房间卡片内清除选中状态，没有全局处理。当学生在一个房间选了床位后，再去另一个房间选床位时，之前的选中状态不会被清除，两个房间的"选定此床位"按钮都会被激活

**解决方案**：修改 `selectBed` 函数，在选择新床位前先清除所有房间的选中状态和按钮激活状态

```javascript
// 修复前
function selectBed(btn, bedId) {
    var card = btn.closest('.room-card');
    var beds = card.querySelectorAll('.bed-btn:not(.occupied)');
    beds.forEach(function(b) { b.classList.remove('selected'); });
    // ... 只在当前卡片内操作
}

// 修复后
function selectBed(btn, bedId) {
    // 清除所有房间的床位选中状态
    document.querySelectorAll('.bed-btn.selected').forEach(function(b) {
        b.classList.remove('selected');
    });
    // 重置所有房间的"选定此床位"按钮
    document.querySelectorAll('.grab-btn.active').forEach(function(b) {
        b.disabled = true;
        b.classList.remove('active');
        b.classList.add('disabled');
    });
    // ... 然后设置当前选中
}
```

**文件**：`student/grab-dorm.aspx`

**教训**：多房间选择场景需要全局状态管理，不能只在当前容器内处理选中状态。
