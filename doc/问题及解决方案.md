# ASP.NET 问题及解决方案

## 问题描述

在创建/修改 ASP.NET Web Forms 页面后，浏览器访问时中文显示为乱码（乱码）。

## 问题原因

### 1. 文件缺少 UTF-8 BOM 标记

UTF-8 BOM（Byte Order Mark）是文件开头的3个字节标记 `EF BB BF`，用于标识文件使用 UTF-8 编码。ASP.NET 默认依赖 BOM 来识别文件编码。

**检查方法：**
```powershell
# 检查文件前3个字节
[System.IO.File]::ReadAllBytes('文件路径')[0..2] -join ','
# 正常应返回: 239,187,191 (即 EF BB BF)
# 如果返回 60,37,64 则是 `<%@`，说明没有 BOM
```

### 2. Page 指令缺少 ResponseEncoding 属性

ASPX 页面的 `@ Page` 指令需要显式指定 `ResponseEncoding="utf-8"`。

### 3. web.config 缺少全球化编码配置

`web.config` 中的 `<system.web>` 节点需要配置 `<globalization>` 元素来指定请求和响应的编码。

---

## 解决方案

### 步骤一：修改 web.config

在 `<system.web>` 节点中添加 `<globalization>` 配置：

```xml
<system.web>
  <compilation debug="true" targetFramework="4.8" />
  <httpRuntime targetFramework="4.8" />
  <globalization requestEncoding="utf-8" responseEncoding="utf-8" fileEncoding="utf-8" />
  <pages validateRequest="false" />
</system.web>
```

### 步骤二：为 Page 指令添加 ResponseEncoding

在每个 `.aspx` 文件的 `@ Page` 指令中添加 `ResponseEncoding="utf-8"`：

```aspx
<%@ Page Language="C#" AutoEventWireup="true" CodeFile="xxx.aspx.cs" Inherits="xxx" ResponseEncoding="utf-8" %>
```

> **注意**：`@ Master` 指令**不支持** `ResponseEncoding` 属性，只能在 `@ Page` 中使用。母版页的编码通过 BOM 和 web.config 的 `<globalization>` 来保证。

### 步骤三：为所有文件添加 UTF-8 BOM

使用 PowerShell 批量添加 BOM：

```powershell
$files = @(
    'path\to\file1.aspx',
    'path\to\file2.aspx'
)
foreach ($f in $files) {
    $content = [System.IO.File]::ReadAllText($f, [System.Text.Encoding]::UTF8)
    $utf8bom = New-Object System.Text.UTF8Encoding $true
    [System.IO.File]::WriteAllText($f, $content, $utf8bom)
}
```

或者批量处理目录下所有 `.aspx` 文件：

```powershell
$files = Get-ChildItem -Path '项目路径' -Include '*.aspx','*.aspx.cs','*.master','*.master.cs' -Recurse
foreach ($f in $files) {
    $bytes = [System.IO.File]::ReadAllBytes($f.FullName)
    if ($bytes.Length -ge 3) {
        if ($bytes[0] -eq 239 -and $bytes[1] -eq 187 -and $bytes[2] -eq 191) {
            Write-Host "$($f.Name) 已有 BOM"
        } else {
            Write-Host "$($f.Name) 缺少 BOM，正在添加..."
            $content = [System.IO.File]::ReadAllText($f.FullName, [System.Text.Encoding]::UTF8)
            $utf8bom = New-Object System.Text.UTF8Encoding $true
            [System.IO.File]::WriteAllText($f.FullName, $content, $utf8bom)
        }
    }
}
```

## 验证方法

### 1. 验证文件 BOM
```powershell
[System.IO.File]::ReadAllBytes('文件路径')[0..2] -join ','
# 应返回: 239,187,191
```

### 2. 验证 Page 指令
```powershell
Get-Content '文件路径' -First 1 -Encoding UTF8
# 应包含 ResponseEncoding="utf-8"
```

### 3. 浏览器验证
- 清除浏览器缓存
- 重新访问页面
- 检查中文是否正常显示

---

## 注意事项

1. **BOM 的作用**：ASP.NET 运行时依赖 BOM 来确定文件编码，缺少 BOM 可能导致中文乱码
2. **CSS 文件**：CSS 文件也可能需要 BOM，特别是包含中文注释时
3. **特殊文件**：`checkcode.aspx`（验证码图片）和 `Default.aspx`（重定向）通常不需要 `ResponseEncoding`
4. **新建文件**：在 Visual Studio 中新建文件时，默认会添加 BOM，但通过代码工具创建的文件可能缺少 BOM
